⇄ 整合 API 文件
系統整合 API 參考

本系統建構於 Supabase（PostgreSQL + PostgREST），所有資料表皆自動對外提供標準 REST API，可供下列外部系統整合。所有系統共用同一組 設備編號 / 案件編號 / 設備履歷，避免重複建檔，形成完整的設備生命週期管理。

一、連線資訊

項目值
API 基礎網址https://qztffronusdhgxhjjubt.supabase.co/rest/v1
驗證方式HTTP Header：apikey + Authorization: Bearer <JWT>
資料格式JSON
⚠ 外部系統整合請使用獨立的 service_role 或為整合對象建立專屬 API 金鑰與 RLS 政策，勿外流前端 anon 金鑰。

四、可整合系統對應

#外部系統共用資料 / 整合方式
1電子交接簿系統共用 案件編號 / 設備編號；交接時帶入未結案件（handover_cases ⇄ repair_requests）
2巡檢管理系統inspection_records 異常可直接轉報修（source=inspection）
3預防保養系統依 equipment 履歷與保固到期排程；寫入 maintenance_orders
4設備管理 CMMS共用 equipment 主檔與全生命週期履歷
5人員排班系統users / 班別；派工時讀取值班技師
6門禁系統以 user/設備位置 比對進出授權（讀 users、locations）
7ERP 系統cost_records 維修成本回拋；資產編號對應 equipment
8採購系統等待料件狀態觸發請購；零件對應 cost_records(parts)
9倉儲庫存系統更換零件扣庫；parts_used 對應庫存品項
10BI 戰情儀表板直接讀取本系統 REST 或 SQL view 供 BI 彙整

五、Webhook / 事件（預留）

可於 Supabase 建立 Database Webhooks 或 Edge Functions，於 repair_requests / maintenance_orders 異動時推送事件至外部系統（如 ERP、採購、LINE/Email）。本系統 notifications 表為站內通知來源，外部推播（Email / LINE / 手機推播）由 Edge Function 串接。

📌 所有資料表已啟用 RLS。正式對外整合前，請為每個整合對象建立最小權限的 Policy 與專屬金鑰。
